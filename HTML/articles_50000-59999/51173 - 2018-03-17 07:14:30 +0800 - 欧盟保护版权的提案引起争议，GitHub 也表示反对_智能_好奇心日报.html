从 2015 年开始，欧盟提出了数十项提案、政策推进所谓的数字单一市场（DSM）策略。

该策略的目标是改善欧盟国家内的电商服务、版权保护机制、简化增值税等多个方面，以促进经济增长。

欧盟计划在 2018 年完成立法的所有提案中，其中一项提案，2016 年的《欧盟版权指令（EU Copyright Directive）》，在过去几年吸引了大量的争议。该提案的覆盖范畴是新闻出版和数字内容。

其中引起大量争议的是第 13 条，对那些存储和开放大量作品和其他用户上传资料的信息服务提供商提出要求，要求这些公司采取措施确保版权协议的有效性，例如对上传内容（主要针对音乐、视频）进行审查。

欧盟称，互联网成为了大量版权内容的分发渠道，版权方有时候会因为盗版无法获得应有的收益。也就是说，第 13 条内容的设立，是为了保护版权方的收益，打击盗版。

指令是欧盟的一种立法，要求欧盟成员国达成某种目标，但不限制具体实施方式，由欧盟议会和理事会审议批准或否决。欧盟各国有可能会采取更多的监管手段。

包括知名的代码托管平台 GitHub，两家提倡自由软件、开放的非盈利机构 Free Software Foundation Europe 和 Open Forum Europe 在内的科技公司都反对该提案。

GitHub 是最新一家，它还计划在本月 20 日的欧盟议会早餐会上，提出意见。

GitHub 在官方博客上发布文章称，如果欧盟正式立法，要求 GitHub 等平台对代码进行审查，一方面这种审查与欧盟保护隐私、现有的法律相悖，还会给中小开发商带来更复杂的流程机制、甚至可能是诉讼。

另外一方面，这在软件开发上很难施行。GitHub 解释称，软件开发者创造的代码作为一种有版权的内容，部分开发者选择采用开源方式将代码分享出去，允许其他开发者在此基础上进一步改善，这意味着几串代码上很可能有不同的开源授权。审查的难度会相当耗费资源，也降低效率。

代码被认为是自由言论的表达，该论点被多次引用过。数字民权组织电子前线基金会（EFF）在 1987 年帮 UC 伯克利的大学生伯恩斯坦打官司时，当时的辩护律师辛迪·科恩（Cindy Cohn ）就提出：代码也是言论的一种，应当受到言论自由的保护。

在 2016 年与美国联邦调查局的争议中，苹果公司同样引用了这条论点，认为代码应当与言论享有同样的权利。

这条提案对于 GitHub 来说，意味着更多的麻烦。欧盟议会成员 Julia Reda 认为，该提案可能会迫使 GitHub 向每一位开发者都谋求开源授权，这会导致 GitHub 平台在欧洲几乎没法发展。

更早向欧盟提出反对意见的，是在去年 9 月份，Free Software Foundation Europe 和 Open Forum Europe 两家非盈利机构发起了请愿活动 Save Code Share，写了公开信反对欧盟该提案。

但抛开这些科技公司、机构对于代码的观点外，欧盟的版权提案在实际操作中仍然存在很大的问题，这也是欧盟提案更大的问题。

盗版的生意只有成长到足够的规模后，才会影响到版权方的利益。但足够规模的盗版生意，也不会再依赖于 GitHub 平台来负责托管代码。欧盟再从 GitHub 等平台上施行盗版数据的审查、监管，也就没有什么意义了。
