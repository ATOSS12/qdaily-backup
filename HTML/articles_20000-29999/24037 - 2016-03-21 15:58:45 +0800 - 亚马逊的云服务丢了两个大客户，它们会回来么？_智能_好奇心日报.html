亚马逊的云服务丢了两个大客户，它们会回来么？_智能_好奇心日报

赵长存 2016-03-21 15:58:45

是接一根电缆，还是自己建发电机，这是一个问题

亚马逊最赚钱的云计算服务 AWS ，最近遭遇了点小挫折。

据 CRN 的消息，苹果公司最近已与 Google 签署了一项 4-6 亿美元的协议，用以购买 Google 的云服务 GCP （Google Cloud Platform）。目前尚不清楚这笔费用是按年支付，还是根据存储容量支付。

CRN 还援引知情人士的话称，苹果已于去年年末开始减少在亚马逊云服务 AWS 上的花销。原本 AWS 主要被用于 iCloud 的数据存储。

按摩根士坦利分析师 Brian Nowak 一月时的说法，苹果大约每年要在数据存储方面花费 10 亿美元，并预计未来两年在 AWS 的年花费将达到 10.5-11.8 亿美元。这也意味着，Google 已经分走了这单生意近一半的钱。

GCP  的报价更低可能是苹果做出这次选择的原因。该平台在今年一月降价后，就称自己同等性能的产品，比 AWS 便宜 15.7%-41.7%。

但这并不意味这 Google 能拿下苹果全部的云存储订单。目前，苹果除了 AWS 、GCP 外，同时还在使用微软的 Azure 云服务，并且还在建设自己的数据中心。去年，其在美国亚利桑那州、爱尔兰和丹麦的 3 座数据中心就已在规划建设中，前后预期投入 20 亿美元和 17 亿欧元。

最终，苹果可能要用自家数据中心里的服务器来提供 iCloud 服务。

最近放弃 AWS 并自建数据中心的也不止苹果一个。

本月早些时候，Dropbox 在前官方博客上宣布，要把用户数据从亚马逊迁移到自己的数据中心，目前进度已完成 90%。而在本月初，Dropbox 还宣布自己的用户数已经突破了 5 亿，且这一数字还在持续增长中。

由于云服务，是将把数据的运算和存储能力交给专门服务商；企业需要时，即可灵活调整资源配置，而无需预先投入资金和人力去建设数据中心。云计算常被类比为像供电公司提供电力一样提供计算和存储能力。Netflix、Dropbox 都是完全依靠 AWS 崛起的创业公司。

然而，对于用户已成规模的公司，例如苹果和 Dropbox，也都会有自建数据中心，在长期的经营中收回成本的诱惑——毕竟亚马逊总得为自己留一些利润。

这样的尝试不一定成功。2011 年社交游戏开发商 Zynga 就选择了把数据从 AWS 搬离到自己的服务器上。但由于缺乏足够的规模，进而影响了成本的控制，2015 年这家公司又把数据重新放回到 AWS。至于之前向自建数据中心投入的 1.5 亿美元，看起来是打了水漂。
