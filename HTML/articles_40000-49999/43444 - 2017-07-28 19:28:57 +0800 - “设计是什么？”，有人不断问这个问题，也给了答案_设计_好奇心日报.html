“设计是什么？”，有人不断问这个问题，也给了答案_设计_好奇心日报

曾梦龙2017-07-28 19:28:57

“毕加索曾经说过，绘画是一个减法的过程。我觉得，这句话的意思是，你必须先有东西可以减掉。这就是我们从最为复杂的部分着手的原因之一。但最后的成品必须非常简洁。这很困难，对任何人都是如此。”

作者简介：

迈克尔·克罗格（Michael Kroeger，著）当今美国乃至全球，最杰出的 LOGO 设计师、思想家及设计教育家。 1993 至 1996 年间曾任美国平面设计协会（AIGA）凤凰城支部主席。

王敏（编著），中央美术学院长江学者特聘教授，博士生导师，中央美术学院学术委员会副主任。曾任中央美院设计学院院长、香港理工大学设计学院讲座教授、国际平面设计师协会(AGI)中国区主席、 AGI 执行理事、国际平面设计联合会副主席。

书籍摘录：

对话二（节选）

PR =保罗·兰德

SA =史蒂夫·艾特 （Steve Ater，亚利桑那州立大学前平面设计系副教授）

STU =学生

PR 你们认为设计是什么？谁要回答这个问题？

STU 沟通交流能够创造因果之间的联系。

PR 这种说法也对，但是不能给我们带来任何帮助。还有谁要回答？

STU 沟通交流就是因果关系。

PR 沟通交流永远是一种因果关系。它或者让你进入睡眠，或者影响你做其他事。再试试。

STU 藉由熟练的操作，设计能够引导你去其他地方。

PR 这种解释好多了，它能引导你去其他某个地方。

STU 设计是二维的。

PR 为什么设计必须是二维的或三维的？为什么设计能是任何东西？所以，熟练的操作是设计的构成部分——至少我们知道熟练的操作是一种必要。那么，我们是在操作什么东西呢？还有别的回答吗？

STU 形式与内容。

PR 设计是对形式和内容的操纵与控制。有了这样一种定义，你就知道自己在干什么，将要走向何处了。因此，我安心坐下来，开始操作。那么，操作又意味着什么呢？你在做什么？你所经历的过程正是操作的内容，就是你正在做的事情。

内容就是思想或主题。形式则是你用以处理或实现这个主题的方式或方法。怎样处理这个主题呢？应该用颜色吗？应该用黑色或白色吗？应该大一点还是小一点？应该做成三维的还是二维的呢？要用时髦流行的素材，还是严肃的素材呢？要用博多尼活字字体（Bodoni）或是巴斯克维尔字体（Baskerville）呢？

这些都是你要问的问题，也是设计的操作面的组成部分。因此，在讨论一个主题之前，先行界定讨论的内容颇为必要。大多数时候，大家都在谈论设计，但没人了解主题到底是什么，甚至没人想过此事。有些人认为，设计就是他们在领带、浴室的壁纸或地毯花纹上看到的东西。这便是人们对于“设计是什么”这个问题的一般性理解。但其实不然，这并不是设计，只是设计过程的构成部分，仅仅是装饰而已。这就是大多数人界定设计的方式，也是外行人给设计下的定义。我觉得，设计是一个不幸的字眼，但尽管如此，设计仍与我们如影随形。时间退回到前文艺复兴时期，当时，兼具艺术家与建筑师双重身份的乔治·瓦萨里（Giorgio Vasari）曾说，设计是所有艺术，包括绘画、舞蹈、雕塑和写作的基础，也就是说，设计是所有艺术门类的基础。设计是所有艺术领域中形式与内容的巧妙结合。

因此，设计以及平面设计与绘画中的设计其实并无两样。如果按照这个观点得出的自然结论来看，你就能确定，设计与绘画或者设计与雕塑，其实并无差别。假如这里在座的各位有谁是画家的话，我非常肯定他们会强烈反对我的观点，但那并不重要。带个画家进来，如果你们认识谁是画家的话，就带他进来。

STU 我会画画，但我完全认同你的观点。

PR 很好，那我现在就可以离开了。

STU 同样的形式、同样的色彩，乃至同样的问题。

PR 如果你是一个差劲的画家，那你就是个差劲的设计师，对吗？

STU 我希望不是这样。

PR 我是说假如。好，我想我们已经明白了设计到底是什么。还有另外一种定义，即设计是一种关系系统，绘画也是如此。这里所谓的关系，涵盖了一个问题的所有面向，指的是你和画布之间的关系，你和美工刀、橡皮擦或画笔之间的关系，同时也包括构成设计的各个元素之间的关系，无论是黑与白或者线条与色块。

同时，设计也是一种比例系统，即尺寸或大小之间的关系。如果沿着“关系”这个角度来思考，我能这样滔滔不绝讲上一整天，你们当然也可以。这类关系无穷无尽。这正是设计如此难以成功的原因之一，因为你所做的一切都有犯错的无尽可能。设计是一个从繁到简的过程。繁复的部分充满了各式各样的可怕问题。尝试评价和权衡所有的问题，然后把它们予以简化的处理，这是非常困难的工作。

毕加索曾经说过，绘画是一个减法的过程。我觉得，这句话的意思是，你必须先有东西可以减掉。这就是我们从最为复杂的部分着手的原因之一。但最后的成品必须非常简洁。这很困难，对任何人都是如此。这堂课到此结束。

SA 我想问一个问题：对于那些在大学或高中教授平面设计的老师而言，哪些方面是研究的重点？我们学习的重点又是什么？

PR 我觉得，了解情况非常重要。重点在于知道你在做什么，在于你能够界定你的研究和学习主题。就你的情况而言，重点在了解平面设计史和设计史，两者都是相同的历史。接下来的建议可能有点偏离我们的主题。重点是要了解美学，也就是针对形式与内容的研究，我们也将美学纳入设计的范畴。美学与设计殊途同归，本质相同。美学是对设计的研究，也是对关系的研究，极其复杂。

我一直建议大家阅读，但基本没有人愿意读书。即便你读的是约翰·杜威的《艺术即经验》，就算你已经将此书从图书馆借出来了，但当读了第一句之后，你就会把书放下，完全忘了读书这档子事。如果你不想读那本书，可以试试门罗·比厄斯利（Monroe Beardsley）的《美学：批评哲学中的问题》（Aesthetics: Problems in the Philosophy of Criticism, 1958），这本书大约有200页。还有几本篇幅更长的书也可以读读，比如黑格尔（Hegel）的《美学：美术讲稿》（Aesthetics: Lectures on Fine Art, ca. 1820），这本书超过了1000页。

但我真的认为，除非读过《艺术即经验》，否则不算接受了设计教育。我在这里给你们提个醒，你们很可能在读了第一句之后就会将书放下。但是，那些读完全书的人，一定会收获满满，而且对作者充满了感激。我这样算是回答了你的问题吗？

SA 如果我们打算开始学习设计，那么，初学者应该学习哪些方面？重点是什么？他们应该使用百利金这类材料，还有裁切色纸吗？

PR 当然。那是必须的。运用双手非常重要，那是将你和一头牛或电脑操作员区别开来的方式。我不想给你们留下我很反感电脑的印象，实际上，我刚刚著完了一本书，这本书大约有两百页，每一页都是在电脑上完成的，当然不是我自己操作的。电脑操作是由一个懂行的人做的，我太老了，做不了这个事情。每次我开个头，你们就得做这做那。

当耶鲁大学最初引进电脑的时候，我认为那是一场灾难。这并不是因为我很反感电脑，事实上，我并不反对。我认为电脑是一种难以置信，甚至令人震惊的机器。但是，电脑本身的特质，也就是那种诱人的特质，正好是它的害处所在。尤其是对那些必须学习设计基础知识的初学者而言，电脑的弊端暴露无遗。如果你必须阅读杜威的著作，又要进行电脑操作，那么这就是一个非常麻烦的问题了。你必须判定哪个更为重要。如果觉得电脑重要一些，恰好你又是一个非常棒的电脑操作员的话，那你大概会在某家广告公司的电脑前忙忙碌碌地度过余生。因为你的技术将会日渐娴熟、与日俱进，你对老板的作用也越来越大，所以你永远不会再做设计了。

有一个给我干活的学生毕业于罗德岛设计学院，实际上，他就是那个在电脑上把我的书制作完成的人。现在，我帮他找了一份工作，在纽约一家非常大的广告公司，这是那种他打算干到退休的工作。他当初过来给我干活，本来意在学做设计。所以，我觉得，正确对待电脑的角色，然后将其摆在恰如其分的位置，非常非常重要。

事实上，你会用电脑，能够操作所有的系统，会用夸克（Quark）和其他所有软件，这与作为设计师知道自己在做什么相比，显得微不足道。因为电脑没法教你怎样才能成为一名设计师。根本不可能！你们知道，当打字机被发明出来的时候，它最伟大的成就，就是摧毁了手写字。如果你们看过打字机发明之前的早期手稿，你们就会明白我在说什么。虽然我不是预言家诺斯特拉达姆斯（Nostradamus） ，但我还是认为，因为电脑的出现，某种类似的命运轨迹也会发生在艺术领域。我觉得这种关系是有害的，不利于艺术的发展。我的观点可能是错的，但这的确是我的想法。

STU 我们谈论了过程，也是我们设计的方式。

PR 这与设计的定义毫无关系，但你或者其他人在这个创造性的领域中如何工作呢？这就涉及具体的过程。著有《思想的艺术》（Art of Thought, 1926）的格拉哈姆·瓦拉斯（Graham Wallas）是个绝顶聪明的人，他发明了一套概念体系，引导我们如何构思想法。比如，先调查问题的所有面向，理出粗略或精细的梗概，然后忘掉这个问题，总之把它抛到脑后就行。这是有关过程的第一部分，我们可以称之为准备阶段。

有关过程的第二部分是酝酿阶段。将问题忘掉之后，让其繁殖孵化，让它在你的脑海中慢慢熬煮。这可不是我发明的，某个非常聪明的家伙发现这些事情的运作方式就是这样。我知道他说的都对，非常适合我的所思所想。假如我要做某件事情，我会先找到有关此事的各种问题，然后将这些问题抛到脑后，隔天或隔上一个礼拜再来重新看待，然后就会有些不一样的事情发生。所以，酝酿阶段非常重要，忘记一个星期，哪怕一天也好，或者随便多久都行。你掌握时间，所以你能决定。

有关过程的第三个方面是豁朗阶段，问题完整浮现出来。你知道，当你等待了一周之后，然后突然之间，豁然开朗，你有了想法。这时，你会将想法记录下来，看看这个想法是否与你想要做的事情一致。等你将所有的想法记录下来之后，你要仔细观察，并且进行评估。看看这些想法是否可行，人们会不会接受，或者你满不满意。

这就是设计的过程，或者说创造性的过程。从一个问题开始，忘掉这个问题，让问题显现自身或让解决方案自己浮现出来，然后重新评价这个问题。这就是你们未来一直要做的事情。
