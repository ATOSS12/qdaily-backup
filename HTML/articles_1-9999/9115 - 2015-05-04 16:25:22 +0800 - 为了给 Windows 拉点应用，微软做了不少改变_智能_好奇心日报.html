为了给 Windows 拉点应用，微软做了不少改变_智能_好奇心日报

唐怡园2015-05-04 16:25:22

面对 Windows 在移动世界的衰落，微软调整了它对开发者的姿态

昨天你朋友圈可能被一群晒年轻的人刷屏了。这套刷脸工具并非个人创业项目，而是两位微软工程师上月底在其开发者大会上发布的网页应用 how-old.net，目的是宣传自己的图像识别引擎以及背后能抗得住访问高峰的云计算平台 Azure。

遗憾的是，Windows 或者 Windows Phone 平台上很少有应用能火成 how－old.net 这样。截至 2014 年 12 月，根据微软官方公布的数据显示， Windows Store 和 Windows 的应用数量加起来 5.25 万，而 iOS 和 Android 平台的应用数量都超过 100 万。

Windows Store 应用太少，自然没什么用户。不少公司也已经不再开发该平台应用，或者开发了也不更新。这不奇怪。大多数公司不会为了占有率不到 4％ 的市场，组建开发团队。毕竟，现在程序员人工成本那么高。

然而，开发者少不仅仅因为 Windows Phone 卖不出去。市场占有率高的 Windows 同样争取不到开发者。iPhone 依然是目前最赚钱的平台 iPhone，而 iPhone 开发应用需要装有 X Code 的 Mac。这导致相当一部分程序员开始用 Mac。

现在微软做了些改变，希望自己的平台能吸引更多的开发者。

一个 Mac 上能用的编程工具

既然那么多程序员已经习惯用 Mac 了，那就开发一个在 Mac 上能用的编程工具，而不是反过来劝说他们用 Windows。这是目前微软的策略。

微软 build  2015 大会上推出了代码编辑器 Visual Studio Code 。谈不上新，但这是 Visual Studio 第一次支持非 windows 平台。

让 Visual Studio 变得强大的 .NET 代码库也在这次大会上宣布开源。文本编辑器样式很多，开发者都有自己的审美和编写偏好，但背后的代码库和插件才是精髓，令人追随的。虽然 Visual Studio Code 暂时没有更多智能的调取插件，但.NET 在 Github 全面开源以后，你在任何操作系统平台上都能调用。

把 iOS 和 Android 应用移植到 Windows 10

Windows Store 的应用数量起不来，微软当时想到的办法之一就是从别的平台移植过来。最开始移植一个 Windows phone 版本应用的工作量和重写一个差不多。后来，微软一直想让这事儿变得简单点。为此，它开源过一部分 .NET 代码，减少移植的代码量。现在 Project Astoria 项目更像是让 Android 变成 Windows phone 的一个子系统。就像你在 Mac 系统下装了个 Windows 的虚拟机。但作为通用应用，它们暂时没能支持特殊的 Android API。

而移植 iOS 应用的 Project Islandwood 主要通过编写环境的同步来简化流程，比如将 Xcode 项目导入 Visual Studio 环境中，尽量保留 Xcode 构建的 iOS 应用代码等等。Candy Crash 就是一个成功实例。

不再有 Windows Phone

Windows 10 要统一手机、电脑、平板，甚至是游戏机 Xbox。在这之前，Windows 和 Windows Store 是分别针对电脑/平板和手机的两套生态。尽管平板和手机都是触屏，但开发者却需要在之前几年里支持两套开发接口、适配两个应用商店。

Windows 10 粘上了这不必要的裂缝。现在，通过 continuum，你可以在电脑上全屏使用原生手机应用。

应用在设备间通用了，移动端仍是重点。微软想让运营商帮着一块卖手机应用，目前它们已经找到 90 个合作伙伴。以后买应用，不用通过信用卡，可以直接充话费。

把赚钱利器 Office 365 开放出来

微软最好用的办公软件要变成平台了，包括 Word，Excel，PowerPoint 以及 Outlook。Office 2016 全面开放第三方接口以后，比如 Uber 和 linkedin 已经接入，将会有更多有趣的功能。其中，Uber 会根据你 OutLook 的行程安排推送订单。

目前，微软拥有 15 亿办公软件用户。说不好 office 365 就会成为微软除了搜索引擎、浏览器之外，又个大流量入口。基于 Office 365 开发一个应用也许会有不错的效果。

把 Chrome 和 Firefox 插件搬到 Edge

IE终于被干掉了。微软的新浏览器 Microsoft Edge 从头来开始。外观设计上看起来有点像 Chrome 的 Edge ，融合了智能语音助手 Cortana，让搜索变得更省事。此外，为了更好的移动端体验，你可以直接在网页上涂涂画画。

作为一个从头开始的浏览器，它将先直接兼容 Chrome 和 Firefox 上的大量插件。而作为开发者，在 Edge 上写一个好用的插件也是个不错的想法。


微软的新生态能不能像 how-old.net 一样蹿红，吸引到开发者是第一步。究竟能不能成功？不少开发者对此是兴奋的：这么开放的微软还是头一回见。
