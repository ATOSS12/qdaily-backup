微软要去拉苹果应用了，但这救不了 Windows 手机_智能_好奇心日报

赵长存 2015-08-07 22:43:30

靠移植应用就能吸引开发者和消费者吗？

和 Google 的 Android 及苹果的 iOS 系统抗衡了这么多年，Windows 手机上也只有 34 万应用，与前两者数以百万计的应用相比，还是少的多。

昨日，微软在其官方博客上发布了一个名为 Windows Bridge for iOS 的项目，想要帮开发者把 iOS 的应用更容易地移植到 Windows 系统上来。

目前这个项目已选择使用 MIT 开源许可协议，并将代码放到了开源平台 Github 上。不过，它还处在一个早期的阶段。换而言之，微软只是想借此吸引更多的开发者协助它完成开发而已。



早在今年 4 月的微软开发者大会上，微软就宣布要把网页应用、Android 应用，以及 iOS 应用都带到 Windows 应用商店来，开发者只要把原有的代码做少量修改就能运行在 Windows 10 上了。所以如今的 Windows Bridge for iOS 正是当时名为 Islandwood 的项目。

据微软称，Windows Brigde 将于今秋正式发布。可是到时候，这真的能拯救 Windows 手机吗？其实早些年，微软也多次花费重金吸引开发者为其开发移动应用，但结果大家也都能看得出来。

谈到靠移植应用来拯救平台，不禁让人联想到，2012 年黑莓也曾推出过移植工具，方便 Android 开发者向其移动系统 BBM 10 移植应用，甚至到了去年干脆让黑莓手机直接运行 Android 应用。但如今的黑莓的手机业务，其实也没有什么起色，更是要做起 Android 手机了。

更何况，Windows Bridge for iOS 甚至不能移植苹果最新的 Swift 语言所开发的应用，而只能移植较老的 Objective-C 语言的应用。这意味着，一些新的应用也就未必能方便地移植到 Windows 上了。

当然，靠移植别的平台的应用对于微软可能也没有什么用。毕竟，要拿出点自己独家的应用，才能构成对消费者的吸引力。

要不然，随着像小米 4 这样的 Android 手机也能运行上 Windows 10 系统，为什么我们不干脆直接购买 Android 手机或者 iPhone 呢？
