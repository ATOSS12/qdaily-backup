微软学黑莓，把Android应用移植到Windows手机上_智能_好奇心日报

赖江锟 2015-07-30 14:08:00

同样的事情，黑莓换成微软，结果会怎样？

在最近推出的 Windows 10 Mobile 系统的预览版上，第一次出现了 Android 子系统，允许 Windows 系统运行移植过来的 Android 应用 (非原生)。该 Android 子系统是微软 Astoria 平台的一部分。 

微软在今年 4 月底的 Build 开发者大会上推出了 Astoria 平台，作用是帮助开发者将 Android、iOS、网页应用移植到 Windows 手机系统。利用 Astoria 平台，开发者无需为 Windows 重新开发应用，只要对现有 Android 应用的代码作部分修改，并将 Google 的服务换成微软的集成到应用中，就能快速做出 Windows 手机应用。

微软这么做的目的是为了快速弥补在手机应用上的不足。数据显示，目前 Google Play 和苹果 App Store 分别有 160 万和 150 万个应用，Windows Phone Store 只有 34 万个，差距巨大。

在 Windows 手机上运行 Android 应用，对开发者吸引力如何还不得而知，但消费者对此可能并不感冒。

在 Lumia 1520 上运行包装过的 Android 应用

即使经过包装，它毕竟还是个 Android 应用，在使用体验上无法 100% 保证与 Android 手机相同。这种感觉有些奇怪，你很容易想起在 MacBook 上安装虚拟机使用 Windows，或是在 Android 手机上使用一款 Windows Phone 的主题。为了用 Android 应用，消费者为什么不直接买台原汁原味的 Android 手机呢？

微软如今的努力，黑莓已经做过。和 Windows 手机一样，黑莓也面临着应用匮乏、对开发者没有吸引力的问题。早在 2011 年 3 月底，黑莓便宣布支持 Android 2.3 的应用，开发者只需将 Android 应用的代码进行重新打包、并作些修改，就变成了“黑莓”的手机应用。结果是，接下来三年内，黑莓手机的销量持续下降。

去年 2 月，黑莓更是允许 Android 原生应用直接在其手机上运行。但 Android 应用显然没有为黑莓的屏幕大小和键盘进行优化，黑莓手机运行 Android 应用已经出现许多兼容性方面的问题。

看起来，黑莓的这种做法并未取得成功。到了本月初，黑莓已经要直接做 Android 手机了。

近几年，微软在手机业务上的努力并没有什么起色。这次步黑莓后尘拿 Android 做文章，看起来又是疑云重重。
