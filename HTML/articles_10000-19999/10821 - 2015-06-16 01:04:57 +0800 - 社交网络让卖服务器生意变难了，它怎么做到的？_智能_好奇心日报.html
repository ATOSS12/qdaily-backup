社交网络让卖服务器生意变难了，它怎么做到的？_智能_好奇心日报

唐怡园 2015-06-16 01:04:57

Facebook 靠开放技术，给自己节省了数十亿美元的成本

无论企业，还是作为消费个人的你，都越来越能接收云计算、云存储这些概念了。企业不用再花大价钱去搭服务器，运维的成本也在下降。而至今已经 4 年的 Facebook 服务器开源项目 Open Compute Project（OCP） 提供了另一种思路。

从 2011 年 4 月开始，Facebook 陆续在代码开源社区 Github 上公开了项目的开源接口，包括数据中心、硬件管理、服务器、网络服务、开放桥架、存储功能以及兼容性和交互性（C&I）等等。
社交网络公司通过 OCP，相当于做了一个企业级管理软件的“应用商店”。来自 Facebook 内部的开发人员和平台申请开发者，在不同分支项目中设计功能，形成模块，方便其他开发者使用。
从前像惠普 、思科、IBM 的服务器业务，一般都是靠着帮企业解决所有问题来收取高昂费用。而 Facebook 将这部分技术开源出来了，企业用户可以购买更便宜的硬件自己搭建数据中心。
Facebook  并不生产服务器。根据 OCP 官方博客数据，截至今年，已经有超过 150 家公司加入到这个开源项目中，包括戴尔、思科、惠普、IBM 这些传统的硬件销售强势公司。而从 2011 年开始，服务器市场增长缓慢。而云计算业务也降低了公司自己组建数据中心的理由。

另外，苹果、微软也加入到 OCP 中，Facebook 的硬件平台不再只支持开源系统 Linux ，也包括 OS X 和 Windows。而包括英特尔在内的一部分芯片制造商，开源了处理器的服务器接口。

过去 3 年里为 Facebook 自己省了 20 亿美元。然而，靠省电省钱，才不是 Facebook 的最终目的。OCP 让后端开发架构逐渐变成一门和软件开发类似的生意。服务器开源的姿态先是为 Facebook 在开发者那里赢得了声誉。一批技术人才趋之若鹜。如今，Facebook OCP 的 Github 主页上每个分支项目都有成千上万的收藏量，代码贡献量也不少。
而越来越多金融机构开始对 OCP 感兴趣，包括美国富达投资集团、高盛集团、花旗银行、美国银行等。
