微软本来想把Android应用移植到自家手机上，现在说不玩了_智能_好奇心日报

徐弢2015-11-16 20:17:44

为 Windows 手机开发应用，应用开发者很可能陷入到“应用少－用户少”的死循环。

所有把其它平台的应用移植到自己手机上的努力都失败了，最新的案例来自微软。

根据 Re/code 等多家科技媒体的消息，微软最近停止了一项名为 Project Astoria 的项目，这个项目的目的，就是将移植 Android 应用到 Windows 手机上。

这是微软在今年 4 月份的 Build 开发者大会上推出的项目。开发者只需要修改 Android 应用的部分代码，并切换微软的服务，能比较方便地做出 Windows 手机应用。

这只是微软的应用移植项目的一部分，除了 Android 版本，还有 iPhone 版本，他们统称为 Windows Bridge 项目，想要将其他平台的应用都能不花费大力气就能转移到 Windows 平台上。

在 Lumia 1520 上运行包装过的 Android 应用

但这个项目至少死了一半了。微软官方这么回应这次 Android 应用转移项目关闭：“Astoria 平台还没有准备好”，但离砍掉 Project Astoria 似乎已经不远了：有开发者发现，微软在 9 月之后再没有回应开发者论坛里关于这个项目的问题，另外，最新的 Windows 10 Mobile Insider 版本内也已经移除了 Android 子系统。

即便 Project Astoria 被砍掉，这种结果也丝毫不令人惊讶，而对开发者来说影响几乎为零。根本原因是，开发者没有为微软开发手机应用的根本推动力，如果他们这样做了，很可能会陷入到“应用少—用户少”的死循环中，对他们根本没有好处。

希望借助移植现有应用来扩充移动应用商店的，微软不过是最新的一家，在此之前，还有熟悉的黑莓，因为和 Windows Phone 一样份额微小，所以兼容其他平台应用看起来也是理所当然。

黑莓 Priv 手机

从 2011 年开始，黑莓开始支持修改版的 Android 2.3 应用，随后支持原生应用，今年 11 月份正式发售第一款 Android 手机 Priv。

为了吸引到开发者开发黑莓手机上的应用，黑莓的 CEO 程守宗今年年初甚至亲自撰文，希望“网络中立”能够扩展到内容和应用上，让 iOS 和 Android 的开发者开发 Windows 版本的应用，比如 iMessage 推出黑莓版。

但这些努力，有多少帮忙提升份额的作用？微软关闭部分的项目，就说明了问题。
