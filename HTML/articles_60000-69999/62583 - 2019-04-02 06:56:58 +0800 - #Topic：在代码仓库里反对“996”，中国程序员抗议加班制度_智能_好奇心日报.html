#Topic：在代码仓库里反对“996”，中国程序员抗议加班制度_智能_好奇心日报

任思远2019-04-02 06:56:58

“程序员的命也是命”

据《南华早报》，3 月 26 日，在软件开发者常用的工作和交流平台、用于软件源代码托管的 GitHub 上，有一名名为 996icu 的用户创建了“996.ICU”的代码仓库（repository)。账户名和仓库名的意思是“以 996 的方式工作，在 ICU（重症监护室）里病倒”。

在这个“仓库”的页面上，“996” 被解释为“越来越受欢迎的非官方工作时间表（上午九点到晚上九点，每周六天）”——服务于鼓励“996”的公司通常意味着每周至少工作 60 小时。

用户还在介绍后面加上了一句“程序开发者的命也是命（Developers' lives matter）”，模仿美国对抗种族不平等运动的“黑人的命也是命（Black Lives Matter）”。发起者希望有人不断参与进来，更新中国实行“996”制度公司的黑名单，维护劳动者的权益。

容纳“代码仓库”的软件源代码托管平台 GitHub 是程序开发者常用的网站，它在 2018 年被微软收购，但在 2015 年时就已经有 2800 万注册用户，是世界上最大的代码存放网站。具体而言，一个程序员在这个网站上注册账户之后，就可以在上面创建公开的、或者私人的代码库，把写好的代码保存在上面，用来进行日常工作。

除了聚集工作的程序员外，GitHub 还为他们提供了交流的功能和空间。这首先是因为在这些被托管的代码中，有不少是开源（开放源代码）项目——允许他人下载、修改和分发源代码，这就导致了在 GitHub上工作的 IT 从业者可以在上面看到任何其他用户的开源项目。另外，GitHub 还设置了有点像社交网络、但更专业的交流方式。比如对他人仓库里的内容进行“星标（star）”表示喜欢；以及“拉出请求（pull request）”，对某人的源代码进行修改、并提交给原作者，让作者决定是否同意这项修改。

所有这些都导致了 996ICU 仓库里的内容不是一个人在房间里的空喊，而引来了各地同行的关注。目前这个代码仓库的“星标（star）”数已经接近 14 万次，在 Github 网页的按照热度排列的“探索”页面上占据第一的位置。除此以外，还衍生出了其他社群（也就是代码仓库），例如 955.WLB，里面有一份不断被更新的“白名单”，上面是遵守“955（早九点到下午五点，一共五天）”制度的公司；996.LAW 主要收集仲裁、民事诉讼信息；而 996.TSC是一个创意板块，允许上传关于“996.ICU”的周边文化设计，比如壁纸、头像、海报，还有文化衫，用来使这项活动传播出去。

目前“仓库”的内容已经有了 895 次修改（commits），修改的内容除了更新“黑名单”之外，还有修改语法等小错误、进行中英文翻译、还有为了突出重点来加粗字体的“拯救 996 的配方”。

996.ICU代码仓库的“readme”部分写明了发起活动的目的：希望有人能在“仓库”中更新实行 “996” 制度公司的名单、上传证据，使事实可见；并且起草一份“劳动保护软件许可证”，让它直接作用在程序员们使用的开源软件或代码上——如果被加入了违反劳动法的“黑名单”，公司将不能使用这些代码和软件。996.ICU 希望通过这个方法强制黑名单上的公司平衡工作时间。

在“仓库”的页面中，《中华人民共和国劳动法》的相关法规被列出，并附上了“相关事件”报道，如“京东回应 995 工作制：不会强制要求 但要全情投入”，以及杭州有赞公司在年会宣布“996 工作制”。并标注“按照劳动法规定，996 工作制下只有拿到当前工资的 2.275 倍，才在经济账上不吃亏”。
